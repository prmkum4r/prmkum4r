## About Me

I build **the interface layer between humans and AI** - real-time voice, conversational UI, and the media pipelines underneath them.

Currently **Senior Software Engineer @ Entropik**, where I ship production AI experiences used in live research and moderation workflows.
In parallel, I'm building **CogniKit** - an open-source toolkit for AI-native frontend development.

My work spans **voice AI, conversational interfaces, AI moderation, real-time media pipelines, and AI-native frontend architecture** - the identity is closer to *AI product engineer* than *frontend engineer*. Ownership for me has consistently meant product architecture and UX decisions, not just component code: performance optimization, production deployment, and end-to-end feature delivery.

## Current Build - CogniKit

> *My bet that AI application development needs its own frontend primitives.*

Traditional UI libraries solved web development. AI-native products need something different - reusable abstractions for conversations, streaming, memory, voice, and agent workflows. CogniKit exists to provide those building blocks, so developers can focus on product experience instead of re-wiring AI infrastructure from scratch on every project.

| Package | Status |
|---|---|
| `@cognikit/core` | ✅ Published on npm - first package live |
| React SDK | 🔧 In active development |
| Conversations primitives | 🔧 Planned |
| Voice primitives | 🔧 Planned |
| Agents / Memory | 🔧 Planned |
| AI UI Components | 🔧 Planned |

**The bet:** most teams building AI products are re-solving the same problems - streaming state, conversation history, voice pipelines - from zero. CogniKit is an early but real step toward a shared, reusable layer for that instead of everyone reinventing it in-house.
